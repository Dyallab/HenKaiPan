SQL Injection in User Login

ID: VIG-10492 calendar_today Oct 24, 2024

Overview

The login endpoint at /api/v1/auth/login is vulnerable to SQL injection through the username parameter. An attacker could bypass authentication or extract sensitive database contents.

Remediation Steps

1
Use Prepared Statements

Replace dynamic SQL string concatenation with parameterized queries using your ORM or native drivers.

2
Input Validation

Implement strict validation on the username field to allow only expected characters (e.g., alphanumeric).

3
WAF Update

Ensure the Web Application Firewall has the latest SQLi signature patterns enabled for the auth path.

Code Evidence

- query = "SELECT * FROM users WHERE uname = '" + user + "'"
+ db.execute("SELECT * FROM users WHERE uname = ?", [user])
